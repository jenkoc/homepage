In the second part of the series, we dive into the heart of our journey with COSMO Loop, moving from the idea to the tangible creation of the MVP. The challenge was clear: How do we transform the theoretical framework of COSMO Loop into a functioning tool that lives up to our vision of seamless knowledge sharing, while also upholding the stringent privacy requirements dictated by our NDAs and EU privacy regulations? This post explores the process of designing, developing, and iterating the MVP of COSMO Loop.

Designing the MVP

The cornerstone of COSMO Loop's MVP was to incorporate a set of core functionalities that would serve as the foundation for our vision. The features were carefully chosen to address the primary objectives.

Search Across All DevOps Organizations

The MVP needed to enable search functionality across all DevOps organizations within Cosmo Consult, ensuring that any piece of information could be easily found by our Cosmonauts.

Access Limited to Cosmonauts

Access control was a critical aspect. The system was designed to be accessible to all Cosmonauts but not to external parties, maintaining the integrity and privacy of our internal operations.

Respect for NDAs and Privacy Regulations

Adherence to NDAs and EU privacy restrictions was non-negotiable. The MVP had to ensure that all shared and searchable content complied with these legal and ethical standards.

Development Challenges

Turning these requirements into reality was no small feat, especially considering the vast scale of our tenant, with hundreds of DevOps organizations. The initial idea of direct queries to the DevOps API proved to be impractical and inefficient. This led us to find an alternative approach.

Getting Access to all DevOps Orgaizations

Even if a DevOps Organization is connected to a specific AD, it does not include any access rights. So there is no user or way to access all organizations with a single user by default.

Introduction of an ETL Process

To manage the sheer volume of data, we develop an Extract, Transform, Load (ETL) process. This process involves scanning, exporting, and enriching work items before storing them. This method ensures efficient data handling and organization.

Utilizing Azure AI Services for PII Extraction

Privacy is paramount. To ensure that personally identifiable information (PII) is appropriately handled, we leverage Azure AI services. This allows us to extract and manage PII from work items, aligning with our commitment to privacy and data protection.

Search Functionality with Azure Cognitive Search

For the search component, we chose Azure Cognitive Search. This decision was driven by its powerful indexing and search capabilities, which allowed us to create a robust and responsive search experience.

Technology Stack

The ETL process is written in JavaScript and deployed to several Azure Functions, which also act as the backend for COSMO Loop. For the frontend, we opt for a lightweight React application, utilizing Bootstrap and Fluent UI framework. This choice is made to achieve a "Microsoft-like" UI, providing a familiar and intuitive user interface for our Cosmonauts.

ETL Process

The following diagram outlines the architecture and process flow for the ETL system used in the COSMO Loop.

Azure Function

The process starts with an Azure Function, written in JavaScript, which is responsible for several key operations. The Daily Export is triggered with a timer-trigger to perform daily exports of data. It identifies what has changed since the last export to ensure that only new or updated items are processed. Most of our DevOps Organizations are created with our own tooling (COSMO Alpaca), so that all specific needs will be applied automatically (i.e. process modifications, repos, pre-defined work items, etc.). We added a step to this tooling where a service user will be added to the organization as well. Then we can leverage that service user to access all organizations.

Azure DevOps

Data is extracted from Azure DevOps, specifically from two types of storage within it: Work Items are the basis of COSMO Loop. The export takes all kinds of Work Item Types into account, not just user stories. COSMO Loop also scans the repos of the organization for specific information.

Data Transformation

Back in the Azure Function, the exported data undergoes a transformation where the relevant Entity, Platform, etc. are getting detected. This is based on assumptions and processes at Cosmo Consult. For example, it can detect the Entity related to the Organization based on specific permissions. It can also detect the line of business and used products (Business Central, D635 FSCM D365 CE,...) based on the content of the repositories.

Azure Cognitive Service

The Azure Cognitive Service is utilized for its Personally Identifiable Information (PII) Detection and other capabilities. This is likely integrated with the anonymization step in the Azure Function to ensure that all PII is identified and handled appropriately. Identified PPIs are getting repalced with a placeholder.

Azure Blob Storage

The final transformed and anonymized data is stored in Azure Blob Storage in a JSON format. Azure Blob Storage serves as a scalable and secure cloud storage solution for large amounts of unstructured data.

React App

The React App is hosted on an Azure App Service, which is the client-facing part of COSMO Loop. The React App is composed of several components that interact with the backend services.

  • Search This component allows users to query and retrieve information. The search provides a list of possbile results based on the search query and filters, sorted by relevance.
  • Suggest COSMO Loop is showing suggestions while the user types in the search bar, like i.e. Google does.
  • Lookup With the lookup, a user can select a specific search result and get more detailed information. All details are shown in a card view.
  • Authentication The React App uses @azure/msal-react to provide a single-sign-on authentication experience.
Azure Function

A JavaScript-based Azure Function serves as a serverless backend for the React application. The React App communicates with this Search API to handle search, suggest, and lookup operations. The Search API acts as a middleware between the frontend and Azure Cognitive Search. The Azure Function is secured with out of the box Azure AD Authentication, to ensure that only Cosmo Consult Employees have access. The data used for search operations is indexed in Azure Cognitive Search. The Search API queries this index to fetch the search results.

Feedback and Iteration

With the core components of the MVP in place, we initiated a pilot program with early adopters within Cosmo Consult. The response was quite positive, and traction grew rapidly. The early feedback we received was invaluable, enabling us to refine and improve COSMO Loop iteratively. This phase was crucial, as it allowed us to address any issues and enhance the tool's functionality based on real-world use and user experiences.
